# generate-pr-description

Create a comprehensive pull request description based on the changes in this branch and format it as proper markdown for use in a GitHub PR description.

1. **Summary** - Provide a clear, concise summary of what this PR accomplishes
2. **Changes Made** - List the key changes made in this PR, include both code and non-code changes, and highlight any breaking changes
3. **Testing** - Describe how the changes were tested, include any new test cases added, and note any manual testing performed
4. **Related Issues** - Link to any related issues or tickets, and use closing keywords if this PR resolves issues
5. **Additional Notes** - Include any deployment considerations, follow-up work required, and notes for reviewers

Focus on:
- Clear, concise summary
- Comprehensive change listing
- Testing documentation
- Issue linking and closing
- Deployment and follow-up considerations

Consider both:
- Code changes (what was modified)
- Non-code changes (documentation, configuration, etc.)

Format the output as:
- PR description in markdown format
- Summary section
- Changes made section
- Testing section
- Related issues section
- Additional notes section

Generate comprehensive PR description with proper markdown formatting for GitHub.
